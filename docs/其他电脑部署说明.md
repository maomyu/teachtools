# 其他电脑部署说明

## 包内包含
- `backend/`：后端源码
- `frontend/`：前端源码
- `backend/database/teaching.db`：当前业务数据库
- `data/local/watermarks/handout_watermark.png`：讲义转换水印资源
- `static/`：静态资源

## 建议运行环境
- macOS / Linux
- Python `3.11`
- Node.js `20+`
- LibreOffice

## 首次部署
1. 解压部署包。
2. 进入 `backend/`：
   - `python3.11 -m venv .venv`
   - `source .venv/bin/activate`
   - `pip install -r requirements.txt`
3. 复制环境文件：
   - `cp .env.deploy.example .env`
4. 编辑 `backend/.env`：
   - 填入你自己的 `DASHSCOPE_API_KEY`
   - 其余路径默认可直接使用
5. 进入 `frontend/`：
   - `npm install`

## 启动方式

### 启动后端
在 `backend/` 目录执行：

```bash
source .venv/bin/activate
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 启动前端
在 `frontend/` 目录执行：

```bash
npm run dev -- --host 127.0.0.1 --port 5157
```

## 访问地址
- 前端：`http://127.0.0.1:5157`
- 后端健康检查：`http://127.0.0.1:8000/health`

## 注意事项
- 讲义转换依赖 LibreOffice；如果没安装，会导致 Word 转 PDF 失败。
- 试卷导入、作文/阅读/完形的 AI 分析依赖 `DASHSCOPE_API_KEY`。
- 当前推荐直接用前端开发服务启动。
  原因：前端项目目前仍有几处与完形讲义相关的历史 TypeScript 报错，会影响 `npm run build`，但不影响本地 `npm run dev` 使用。
